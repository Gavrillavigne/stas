Типы Docker сетей:
Драйверы:
1) bridge - контейнеры могут общаться друг с другом внутри одной bridge сети,
    но не могут обратиться к контейнерам из другой bridge сети;
2) host - у контейнеров нет своего IP адреса. Они просто существуют в сети host машины.
    Если хотим обратиться к какому-либо контейнеру, то это можно сделать через прокинутые порты;
3) overlay - для объединения сетей контейнеров на разных docker машинах (хостах);
4) macvlan - для присвоения каждому контейнеру физического MAC-адреса и обращения к нему, как к настоящему
    существующему устройству;
5) null - если вообще не хотим присоединять контейнер к какой-либо сети.

КОМАНДЫ:
docker network inspect php-up - посмотреть настройки docker сети
docker logs --tail 50 --follow --timestamps name_of_container - логи контейнера
docker-compose logs -f --tail=50 name_of_service - логи контейнера
=================================================================================
ipam - управление IP-адресами.
Через ipam можно задать настройки подсети, в которой у хост-машины всегда будет первый номер из подсети.
Например, 192.168.245.1. И к хост-машине можно обращаться из докер-контейнеров по этому адресу.
=================================================================================
КЛАСТЕРИЗАЦИЯ:
docker node ls - список активных нод для данного кластера

docker swarm init --advertise-addr 192.168.0.8 - инициализация manager ноды
docker swarm join --token your_token ip_addr - подключение worker ноды к manager ноде

docker swarm leave - вывести ноду из под управления кластера
docker node inspect node_name - инспектирование ноды
docker node rm node_name - удалить ноду из кластера
